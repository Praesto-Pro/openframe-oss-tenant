Local Django settings configuration for development and Docker environments with security protections disabled and external service integrations.

## Key Components

- **Environment Variable Substitution**: Uses `${VAR}` syntax for dynamic configuration values
- **Security Disabled**: CSRF, session security, and SSL/TLS protections turned off for development
- **Database Configuration**: PostgreSQL connection with environment-based credentials
- **External Services**: Redis caching, NATS messaging, and MeshCentral (disabled)
- **OpenFrame Integration**: Custom mode toggle for OpenFrame platform features
- **Development Features**: Debug mode, CORS allow-all, and API documentation enabled

## Usage Example

```python
# This file is typically loaded by Django's settings system
# Environment variables must be set before Django startup:

import os
os.environ['DJANGO_SEKRET'] = 'your-secret-key'
os.environ['POSTGRES_DB'] = 'tactical_db'
os.environ['POSTGRES_USER'] = 'tactical_user'
os.environ['POSTGRES_PASSWORD'] = 'password123'
os.environ['POSTGRES_HOST'] = 'localhost'
os.environ['POSTGRES_PORT'] = '5432'
os.environ['TACTICAL_DIR'] = '/opt/tactical'
os.environ['APP_HOST'] = 'localhost:3000'
os.environ['ADMINURL'] = 'admin'
os.environ['REDIS_HOST'] = 'localhost'
os.environ['OPENFRAME_MODE'] = 'True'

# Django will automatically load these settings when running
# python manage.py runserver --settings=local_settings
```

**⚠️ Security Warning**: This configuration disables critical security features and should only be used in development environments. Never deploy to production with these settings.
