A cross-platform service for gracefully stopping and killing tool processes with retry logic and pattern-based process detection. The service handles both individual tools and asset-specific processes using platform-specific command line patterns.

## Key Components

- **ToolKillService**: Main service struct providing process termination capabilities
- **stop_tool()**: Stops processes matching a tool ID pattern
- **stop_asset()**: Stops processes for specific asset within a tool
- **stop_installed_tool()**: Stops processes based on InstalledTool configuration
- **stop_tool_by_path()**: Stops processes by executable path matching
- **stop_processes_by_pattern()**: Generic pattern-based process termination
- **Retry Logic**: Graceful shutdown followed by force kill with configurable timeouts
- **Platform Detection**: Windows vs Unix path pattern matching

## Usage Example

```rust
use anyhow::Result;
use crate::services::ToolKillService;
use crate::models::{InstalledTool, InstallationType};

#[tokio::main]
async fn main() -> Result<()> {
    let kill_service = ToolKillService::new();
    
    // Stop a tool by ID
    kill_service.stop_tool("my-tool").await?;
    
    // Stop a specific asset within a tool
    kill_service.stop_asset("asset-123", "my-tool").await?;
    
    // Stop by executable path
    kill_service.stop_tool_by_path("/usr/bin/my-app").await?;
    
    // Stop an installed tool
    let tool = InstalledTool {
        tool_agent_id: "my-tool".to_string(),
        installation_type: InstallationType::Standard,
        executable_path: None,
    };
    kill_service.stop_installed_tool(&tool).await?;
    
    Ok(())
}
```
