A service that manages OpenFrame client updates by downloading, preparing, and launching update processes with comprehensive safety checks and state management.

## Key Components

- **OpenFrameClientUpdateService**: Main service struct with dependency injection for client info, GitHub downloads, and update state services
- **process_update()**: Public entry point that handles update requests with race condition protection via mutex
- **process_update_internal()**: Core update logic with version validation and status management
- **execute_update()**: Orchestrates the complete update flow from download to updater launch
- **create_temp_archive()**: Platform-specific binary preparation (ZIP for Windows, direct file for Unix)
- **parse_version()**: Semantic version parsing with support for v-prefixed and build metadata
- **is_valid_version()**: Basic version format validation

## Usage Example

```rust
use openframe_client_update_service::OpenFrameClientUpdateService;
use openframe_client_update_message::OpenFrameClientUpdateMessage;

// Initialize service with dependencies
let update_service = OpenFrameClientUpdateService::new(
    client_info_service,
    github_download_service,
    update_state_service,
);

// Process update request
let message = OpenFrameClientUpdateMessage {
    version: "v1.2.3".to_string(),
    download_configurations: vec![/* platform configs */],
};

match update_service.process_update(message).await {
    Ok(_) => println!("Update process initiated"),
    Err(e) => eprintln!("Update failed: {}", e),
}
```

The service ensures thread-safe updates, validates semantic versions, manages update states, and handles platform-specific binary preparation before launching the actual updater process.
