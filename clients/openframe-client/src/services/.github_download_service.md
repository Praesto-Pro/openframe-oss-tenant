A service for downloading and extracting GitHub releases with automatic retry logic and CDN fallback support. Handles platform-specific archive formats (ZIP on Windows, tar.gz on Unix) with size validation and timeout protection.

## Key Components

- **GithubDownloadService**: Main service struct with HTTP client and DMG extractor
- **download_and_extract()**: Downloads archives and extracts target binaries with validation
- **download_with_retry()**: Implements retry logic with exponential backoff and jsDelivr CDN fallback
- **extract_from_zip()**: ZIP archive extraction (Windows-specific)
- **extract_from_tar_gz()**: tar.gz archive extraction (Unix-specific)
- **find_config_for_current_os()**: Selects appropriate download configuration for current platform

## Usage Example

```rust
use reqwest::Client;
use crate::github_download_service::GithubDownloadService;
use crate::models::download_configuration::DownloadConfiguration;

// Create service with HTTP client
let client = Client::new();
let dmg_extractor = crate::platform::DmgExtractor::new();
let service = GithubDownloadService::new(client, dmg_extractor);

// Download and extract binary
let config = DownloadConfiguration {
    link: "https://github.com/owner/repo/releases/download/v1.0/tool.zip".to_string(),
    file_name: "tool.zip".to_string(),
    target_file_name: "tool.exe".to_string(),
    // ... other fields
};

let binary_bytes = service.download_and_extract(&config).await?;
```

The service automatically handles GitHub rate limits by falling back to jsDelivr CDN, validates file sizes, and provides platform-specific archive extraction with proper error handling and logging.
