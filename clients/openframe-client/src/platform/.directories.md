Cross-platform directory management module that provides a unified interface for handling platform-specific directory operations across Windows, macOS, and Linux systems. It manages application directories, logging paths, and secure storage locations with proper permissions.

## Key Components

- **DirectoryManager**: Main struct providing unified directory management API
- **DirectoryError**: Comprehensive error type for directory operations
- **Platform functions**: `get_app_support_directory()`, `get_logs_directory()`, `get_secured_directory()`
- **Permission functions**: `set_directory_permissions()`, `set_secured_directory_permissions()`

## Usage Example

```rust
use crate::directories::DirectoryManager;

// Create default manager with platform-specific paths
let manager = DirectoryManager::new();

// Development mode (uses user directories only)
let dev_manager = DirectoryManager::for_development();

// Custom directories
let custom_manager = DirectoryManager::with_custom_dirs(
    PathBuf::from("/custom/logs"),
    PathBuf::from("/custom/app"),
    PathBuf::from("/custom/secured")
);

// Get platform-specific directories
let logs_path = get_logs_directory();
let app_path = get_app_support_directory();

// Set appropriate permissions
set_directory_permissions(&logs_path)?;
set_secured_directory_permissions(&secured_path)?;
```

The module handles platform differences automatically - Windows uses `%ProgramData%`, macOS uses `/Library/`, and Linux uses `/var/lib/` and `/var/log/`. It also supports environment variable overrides (`OPENFRAME_LOG_DIR`) and development mode for non-privileged operations.
