A macOS-specific utility for writing application preferences using the system's `defaults` command, with cross-platform compatibility for non-macOS systems.

## Key Components

- **`write()`** - Writes key-value preferences to a macOS application bundle using `defaults write` command
- **`args_to_pairs()`** - Converts CLI-style arguments into key-value pairs for preference writing
- **Platform-specific compilation** - Full implementation on macOS, no-op stub on other platforms

## Usage Example

```rust
use preferences_writer::{write, args_to_pairs};

// Write preferences directly to a macOS app bundle
let prefs = [
    ("serverUrl", "https://api.example.com"),
    ("enableLogging", "true"),
    ("maxRetries", "3")
];
write("com.example.myapp", prefs)?;

// Convert CLI args to preference pairs
let cli_args = vec![
    "--serverUrl".to_string(),
    "https://api.example.com".to_string(),
    "--devMode".to_string(),
    "--maxRetries".to_string(),
    "5".to_string()
];

let pairs = args_to_pairs(&cli_args);
// Results in: [("serverUrl", "https://api.example.com"), ("devMode", "1"), ("maxRetries", "5")]

write("com.example.myapp", pairs)?;
```

The module automatically handles user context by running `defaults` commands as the console user via `sudo`, ensuring preferences are written to the correct user's domain. On non-macOS platforms, the `write()` function is a no-op that always succeeds.
