This module serves as the main entry point for OpenFrame's system utilities, organizing platform-specific and cross-platform functionality for file operations, permissions, and system management.

## Key Components

**Cross-platform modules:**
- `directories` - Directory management and path utilities
- `dmg_extractor` - DMG file extraction capabilities  
- `file_lock` - File locking detection and management
- `permissions` - Permission checking and capability management
- `uninstall` - Safe directory removal with retry logic
- `update_scripts` - Update script generation and execution
- `updater_launcher` - Application updater launching
- `user_session` - User session management

**Platform-specific modules:**
- `preferences_writer` (macOS only) - System preferences management
- `windows_cleanup` (Windows only) - Windows-specific cleanup operations
- `powershell` (Windows only) - PowerShell execution utilities

**Re-exported types:**
- `DirectoryManager`, `DmgExtractor` - Core utility classes
- `PermissionUtils`, `Permissions` - Permission management
- `LockingProcess` utilities (Windows) - File lock detection
- Error types: `DirectoryError`, `PermissionError`

## Usage Example

```rust
use openframe_system_utils::{
    DirectoryManager, 
    PermissionUtils, 
    Permissions,
    remove_directory_with_retry
};

// Directory management
let dir_manager = DirectoryManager::new()?;
let app_dir = dir_manager.get_app_directory()?;

// Permission checking
let perms = PermissionUtils::check_capabilities()?;
if perms.contains(Permissions::WRITE_FILES) {
    println!("Write permissions available");
}

// Safe directory removal
remove_directory_with_retry(&old_app_dir, 3)?;
```

The module provides a clean interface to system utilities while handling platform differences transparently through conditional compilation.
