Module aggregator for OpenFrame agent communication types, providing a centralized interface for message structures, configuration types, and state management used in agent-server communication.

## Key Components

**Core Message Types:**
- `AgentRegistrationRequest/Response` - Agent registration flow
- `MachineHeartbeatMessage` - Periodic agent status updates  
- `OpenFrameClientUpdateMessage` - Client update notifications
- `ToolAgentUpdateMessage` - Tool-specific agent updates

**Configuration Types:**
- `AgentConfiguration` - Runtime agent settings
- `InitialConfiguration` - Bootstrap configuration
- `DownloadConfiguration` - Package download settings with `InstallationType` enum

**Tool Management:**
- `ToolInstallationResult/Message` - Installation outcomes and progress
- `ToolConnectionMessage` - Tool connectivity status
- `InstalledTool` - Tool metadata with `ToolStatus` enum
- `ToolConnection` - Active tool connections

**State Management:**
- `UpdateState` - System update tracking with `UpdatePhase` enum
- `SessionType` - Tool session categorization
- `AgentTokenResponse` - Authentication tokens

## Usage Example

```rust
use crate::messages::{
    AgentRegistrationRequest, 
    ToolInstallationResult, 
    UpdateState, 
    UpdatePhase
};

// Register a new agent
let registration = AgentRegistrationRequest {
    machine_id: "agent-001".to_string(),
    capabilities: vec!["docker".to_string()],
};

// Track installation progress
let install_result = ToolInstallationResult::success(
    "docker-tool",
    "Installation completed successfully"
);

// Monitor update state
let update_state = UpdateState {
    phase: UpdatePhase::Installing,
    progress: 75,
};
```

This module serves as the single entry point for all OpenFrame agent communication types, ensuring consistent message handling across the distributed system.
