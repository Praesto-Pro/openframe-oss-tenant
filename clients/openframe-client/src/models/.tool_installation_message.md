A comprehensive message structure for coordinating tool installation operations in the Flamingo platform, defining all necessary configuration and metadata for automated tool deployment across different operating systems and session types.

## Key Components

- **`ToolInstallationMessage`** - Main struct containing tool identification, version info, command arguments, and installation configurations
- **`SessionType`** - Enum defining execution contexts (Service, Console, User) for tool installation
- **`Asset`** - Represents downloadable tool assets with OS-specific filename configurations and source locations
- **`LocalFilenameConfig`** - OS-specific filename mapping with current OS detection capabilities
- **`AssetSource`** - Enum for supported download sources (Artifactory, Tool API, GitHub)

## Usage Example

```rust
use tool_installation_message::{ToolInstallationMessage, SessionType, Asset, AssetSource};

// Create a tool installation message
let installation_msg = ToolInstallationMessage {
    tool_agent_id: "agent-123".to_string(),
    tool_id: "docker".to_string(),
    tool_type: "container-runtime".to_string(),
    version: "24.0.0".to_string(),
    reinstall: false,
    session_type: Some(SessionType::Service),
    download_configurations: None,
    installation_command_args: Some(vec!["--quiet".to_string()]),
    uninstallation_command_args: Some(vec!["--purge".to_string()]),
    run_command_args: vec!["run".to_string(), "-d".to_string()],
    tool_agent_id_command_args: None,
    assets: Some(vec![Asset {
        id: "docker-binary".to_string(),
        local_filename_configuration: vec![
            LocalFilenameConfig {
                filename: "docker.exe".to_string(),
                os: "windows".to_string(),
            },
        ],
        source: AssetSource::Github,
        path: Some("/usr/local/bin/docker".to_string()),
        executable: true,
        download_configurations: None,
        version: Some("24.0.0".to_string()),
    }]),
};

// Check if filename config matches current OS
let config = LocalFilenameConfig {
    filename: "docker".to_string(),
    os: "linux".to_string(),
};
let matches = config.matches_current_os();
```
