Configuration structure for managing cross-platform file downloads with OS-specific handling and flexible field mapping through serde aliases.

## Key Components

- **`DownloadConfiguration`** - Main struct defining download parameters with JSON serialization support
- **`is_folder_extraction()`** - Determines if the target requires full archive extraction based on path structure
- **`matches_current_os()`** - Platform detection utility that compares configuration OS with runtime environment

## Usage Example

```rust
use serde_json;

// Parse download configuration from JSON
let config_json = r#"{
    "os": "linux",
    "fileName": "agent-v1.0.tar.gz",
    "agentFileName": "bin/agent",
    "link": "https://releases.example.com/agent-v1.0.tar.gz"
}"#;

let config: DownloadConfiguration = serde_json::from_str(config_json)?;

// Check if this configuration applies to current platform
if config.matches_current_os() {
    println!("Downloading {} for {}", config.file_name, config.os);
    
    // Determine extraction strategy
    if config.is_folder_extraction() {
        println!("Will extract archive to access: {}", config.target_file_name);
    } else {
        println!("Direct file download: {}", config.target_file_name);
    }
}
```

The struct supports flexible JSON field mapping through serde aliases, allowing `agentFileName` or `assetFileName` to map to `target_file_name` for backward compatibility.
