Configuration structure for managing platform-specific software downloads and installations, supporting different operating systems and installation types.

## Key Components

- **`InstallationType`**: Enum defining installation types (`Standard`, `GuiApp`) with SCREAMING_SNAKE_CASE serialization
- **`DownloadConfiguration`**: Main struct containing download metadata including OS, file names, download link, and installation type
- **`is_folder_extraction()`**: Method to determine if the target requires full archive extraction based on path components
- **`matches_current_os()`**: Cross-platform OS detection method that compares configuration OS with current runtime environment

## Usage Example

```rust
use serde_json;

// Create a download configuration
let config = DownloadConfiguration {
    os: "windows".to_string(),
    file_name: "installer.zip".to_string(),
    target_file_name: "bin/agent.exe".to_string(),
    link: "https://releases.example.com/v1.0.0/installer.zip".to_string(),
    installation_type: InstallationType::Standard,
    bundle_id: Some("com.example.agent".to_string()),
};

// Check if extraction is needed
if config.is_folder_extraction() {
    println!("Full archive extraction required");
}

// Validate OS compatibility
if config.matches_current_os() {
    println!("Configuration matches current platform");
}

// Serialize to JSON
let json = serde_json::to_string(&config)?;
```

The struct uses serde field aliases to support multiple JSON field name formats (`agentFileName`, `assetFileName`, `targetFileName`) and provides platform detection for Windows, macOS, and Linux systems.
