Main entry point for the OpenFrame Chat Tauri application that handles configuration management, token authentication, system tray integration, and window lifecycle.

## Key Components

**State Management:**
- `ServerUrlState` - Thread-safe storage for server URL configuration
- `DebugModeState` - Thread-safe storage for debug mode setting
- `TokenState` - Managed by token watcher for authentication tokens

**Tauri Commands:**
- `greet(name)` - Simple greeting command for testing
- `get_token()` - Retrieves current authentication token
- `get_server_url()` - Returns configured server URL
- `get_debug_mode()` - Returns debug mode status

**Core Function:**
- `run()` - Main application entry point with full setup and event handling

## Usage Example

```rust
// Main application entry point
#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    // Application automatically:
    // 1. Reads config from system preferences
    // 2. Sets up token watcher for authentication
    // 3. Creates system tray with show/quit menu
    // 4. Handles window show/hide behavior
    // 5. Manages background/foreground launch modes
}

// Frontend can call these commands:
// get_token() -> Option<String>
// get_server_url() -> Option<String>  
// get_debug_mode() -> bool
```

The application features system tray integration with left-click to show/hide windows, configuration reading from macOS CFPreferences, token file watching with decryption, and graceful window management that hides rather than closes windows when requested.
