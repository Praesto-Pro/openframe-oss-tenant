A cross-platform configuration reader that loads OpenFrame application settings from macOS system preferences or command-line arguments on other platforms.

## Key Components

- **`AppConfig`** - Main configuration struct containing token path, secret, server URL, and debug mode settings
- **`from_preferences()`** - Platform-aware constructor that uses macOS preferences on macOS, CLI args elsewhere  
- **`from_cli_args()`** - Command-line argument parser for Windows/Linux platforms
- **`is_valid()`** - Validates that all required configuration fields are present
- **`macos` module** - macOS-specific utilities for reading from system preferences using `defaults` command

## Usage Example

```rust
use config_reader::AppConfig;

// Load configuration from platform-appropriate source
let config = AppConfig::from_preferences();

// Validate configuration is complete
if config.is_valid() {
    println!("Server URL: {}", config.server_url.unwrap());
    println!("Debug mode: {}", config.debug_mode);
} else {
    eprintln!("Missing required configuration fields");
}

// Manual CLI usage (non-macOS):
// ./app --openframe-token-path /path/to/token --openframe-secret secret123 --serverUrl https://api.example.com --devMode
```

The configuration reader abstracts platform differences, automatically using macOS system preferences (via the `defaults` command with bundle ID `com.openframe.chat`) or parsing CLI arguments on other platforms. All configuration fields are optional during parsing but can be validated using `is_valid()`.
