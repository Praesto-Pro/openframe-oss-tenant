This component provides a comprehensive logs table interface for displaying and managing log entries with filtering, pagination, and modal detail views.

## Key Components

**Main Exports:**
- `LogsTable` - Main table component with forwardRef support
- `LogsTableRef` - Interface for imperative actions (refresh)
- `UILogEntry` - TypeScript interface for transformed log data
- `LogsTableProps` - Component props interface

**Core Features:**
- Cursor-based pagination with server-side data fetching
- Multi-column filtering (status, tool type, organization)
- Real-time search functionality
- Responsive design with column hiding at different breakpoints
- Modal detail view for individual log entries
- Embedded mode for device-specific display

## Usage Example

```typescript
import { LogsTable, type LogsTableRef } from './logs-table'
import { useRef } from 'react'

function LogsPage() {
  const tableRef = useRef<LogsTableRef>(null)

  const handleRefresh = () => {
    tableRef.current?.refresh()
  }

  return (
    <div>
      <button onClick={handleRefresh}>Refresh Logs</button>
      
      {/* Full-featured logs table */}
      <LogsTable ref={tableRef} />
      
      {/* Device-specific embedded table */}
      <LogsTable 
        deviceId="device-123"
        embedded={true}
      />
    </div>
  )
}
```

The component transforms raw log data from the API into a UI-friendly format, handles complex state management for filters and pagination, and provides both standalone and embedded viewing modes. It integrates with OpenFrame's design system components and supports external navigation to detailed log views.
