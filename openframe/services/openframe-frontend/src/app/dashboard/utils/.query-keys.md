A centralized query key factory for React Query hooks used in the dashboard, providing consistent and hierarchical cache keys for various dashboard statistics.

## Key Components

### dashboardQueryKeys
The main query key factory object that exports standardized cache keys for:

- **all**: Root-level cache key for all dashboard queries
- **deviceStats()**: Query key for device statistics data
- **chatStats()**: Query key for chat statistics (SaaS mode only)  
- **orgStats(limit)**: Parameterized query key for organization statistics with limit
- **invalidateAll()**: Helper to invalidate all dashboard-related queries

## Usage Example

```typescript
import { useQuery, useQueryClient } from '@tanstack/react-query'
import { dashboardQueryKeys } from './query-keys'

// Using query keys in hooks
const { data: deviceStats } = useQuery({
  queryKey: dashboardQueryKeys.deviceStats(),
  queryFn: fetchDeviceStats,
})

const { data: orgStats } = useQuery({
  queryKey: dashboardQueryKeys.orgStats(10),
  queryFn: () => fetchOrgStats({ limit: 10 }),
})

// Invalidating queries
const queryClient = useQueryClient()
queryClient.invalidateQueries({
  queryKey: dashboardQueryKeys.invalidateAll()
})
```

The hierarchical structure ensures that invalidating the root `all` key will cascade to all dashboard-related queries, while individual query types can be invalidated independently.
