This is a Next.js page component that renders the main dashboard with dynamic loading and server-side rendering disabled for optimal client-side performance.

## Key Components

- **Dashboard** - Default exported page component that combines layout and dynamic content
- **DashboardContent** - Dynamically imported dashboard content component with SSR disabled
- **AppLayout** - Wrapper component providing consistent application layout
- **DashboardLoading** - Loading component displayed while dashboard content loads
- **dynamic export** - Next.js configuration forcing dynamic rendering

## Usage Example

```typescript
// This page is automatically rendered when users navigate to /dashboard
// The dynamic import ensures the dashboard content is loaded client-side only

// Example of the dynamic loading pattern used:
const DynamicComponent = nextDynamic(
  () => import('./heavy-component'),
  { 
    ssr: false,  // Disable server-side rendering
    loading: () => <LoadingSpinner />  // Show loading state
  }
)

// The force-dynamic export ensures the page is always rendered dynamically:
export const dynamic = 'force-dynamic'
```

The component uses Next.js dynamic imports to code-split the dashboard content, improving initial page load performance. The `ssr: false` option ensures the dashboard renders only on the client side, which is useful for components that depend on browser APIs or user-specific data. The `force-dynamic` export tells Next.js to always render this page dynamically rather than statically.
