Hook for managing real-time NATS subscriptions to Mingo chat dialogs, handling streaming messages, chunk processing, and connection state management. Provides unified subscription management for multiple dialogs with automatic catch-up and state synchronization.

## Key Components

- **useMingoRealtimeSubscription**: Main hook for managing dialog subscriptions and connection state
- **useDialogChunkProcessor**: Per-dialog chunk processing with message streaming and approval handling
- **DialogSubscription**: Individual dialog subscription component with NATS WebSocket integration
- **MINGO_TOPICS**: Supported NATS message types for Mingo subscriptions
- **getApiBaseUrl/getAccessToken**: Utility functions for configuration retrieval

## Usage Example

```typescript
// Basic subscription management
const {
  subscribeToDialog,
  unsubscribeFromDialog,
  getSubscriptionState,
  subscribedDialogs,
  connectionState
} = useMingoRealtimeSubscription('dialog-123', {
  onChunkReceived: (dialogId, chunk, messageType) => {
    console.log('Received chunk for dialog:', dialogId)
  }
})

// Subscribe to a dialog
subscribeToDialog('dialog-456')

// Check subscription state
const state = getSubscriptionState('dialog-456')
console.log('Is connected:', state.isConnected)

// Individual dialog subscription component
<DialogSubscription
  dialogId="dialog-123"
  isActive={true}
  onApprove={(requestId) => handleApproval(requestId)}
  onReject={(requestId) => handleRejection(requestId)}
  approvalStatuses={approvalStates}
/>
```

The hook integrates with the Mingo messages store for state management and provides real-time streaming capabilities with automatic reconnection and catch-up functionality for missed messages.
