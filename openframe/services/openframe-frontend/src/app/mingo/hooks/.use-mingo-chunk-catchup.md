This hook provides a Mingo-specific implementation of chunk catchup functionality for real-time chat dialogs, wrapping the core OpenFrame chunk catchup system with custom API integration.

## Key Components

- **`useMingoChunkCatchup`** - Main hook that manages real-time chunk synchronization for Mingo chat dialogs
- **`fetchChunks`** - API client function that retrieves chat chunks from the backend with optional sequence filtering
- **Type exports** - Re-exports core types (`ChunkData`, `NatsMessageType`, `UseChunkCatchupReturn`) for consistent typing
- **`UseMingoChunkCatchupOptions`** - Configuration interface requiring dialog ID and chunk received callback

## Usage Example

```typescript
import { useMingoChunkCatchup } from './use-mingo-chunk-catchup'

function ChatComponent() {
  const { isConnected, lastChunk, error } = useMingoChunkCatchup({
    dialogId: 'dialog-123',
    onChunkReceived: (chunk, messageType) => {
      console.log('Received chunk:', chunk)
      // Handle new chunk data (append to messages, update UI, etc.)
    }
  })

  if (error) {
    return <div>Error: {error}</div>
  }

  return (
    <div>
      <div>Status: {isConnected ? 'Connected' : 'Disconnected'}</div>
      {lastChunk && <div>Last message: {lastChunk.content}</div>}
    </div>
  )
}
```

The hook automatically handles ADMIN chat type filtering and provides real-time synchronization with graceful error handling for failed API requests.
