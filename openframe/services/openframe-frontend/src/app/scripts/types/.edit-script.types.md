Defines TypeScript schemas and types for script editing functionality using Zod validation, ensuring proper data structure and validation for script configuration forms.

## Key Components

- **`scriptArgumentSchema`** - Zod schema for script arguments with id, key, and value fields
- **`editScriptSchema`** - Main validation schema for script editing with comprehensive validation rules
- **`EditScriptFormData`** - TypeScript type inferred from the edit script schema
- **`CATEGORIES`** - Predefined script categories array
- **`EDIT_SCRIPT_DEFAULT_VALUES`** - Default form values for new script creation

## Usage Example

```typescript
import { editScriptSchema, EditScriptFormData, EDIT_SCRIPT_DEFAULT_VALUES } from './edit-script.types'

// Validate script data
const validateScript = (data: unknown): EditScriptFormData => {
  return editScriptSchema.parse(data)
}

// Create new script with defaults
const newScript: EditScriptFormData = {
  ...EDIT_SCRIPT_DEFAULT_VALUES,
  name: 'System Cleanup',
  script_body: 'Get-ChildItem -Path C:\\temp | Remove-Item -Force',
  description: 'Cleans temporary files from system',
}

// Add script arguments
newScript.args.push({
  id: '1',
  key: 'path',
  value: 'C:\\temp'
})
```

The schema enforces validation rules like minimum timeout of 1 second, maximum of 24 hours, required fields, and platform selection constraints.
