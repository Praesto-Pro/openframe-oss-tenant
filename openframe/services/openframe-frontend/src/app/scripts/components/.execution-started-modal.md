A confirmation modal component that notifies users when a script execution has been initiated and provides options to view logs or dismiss the notification.

## Key Components

- **ExecutionStartedModal** - Main modal component that displays execution confirmation
- **ExecutionStartedModalProps** - Interface defining the modal's props including open state, script name, and callback handlers
- **Modal UI Components** - Utilizes OpenFrame's modal system with header, content, and footer sections

## Usage Example

```typescript
import { ExecutionStartedModal } from './execution-started-modal'
import { useState } from 'react'

function ScriptExecutor() {
  const [isModalOpen, setIsModalOpen] = useState(false)
  const [executedScript, setExecutedScript] = useState('')

  const handleScriptExecution = (scriptName: string) => {
    // Execute script logic here
    setExecutedScript(scriptName)
    setIsModalOpen(true)
  }

  const handleViewLogs = () => {
    // Navigate to logs view
    setIsModalOpen(false)
    // router.push('/logs') or similar navigation
  }

  return (
    <>
      <button onClick={() => handleScriptExecution('backup-script.sh')}>
        Execute Backup Script
      </button>
      
      <ExecutionStartedModal
        isOpen={isModalOpen}
        onClose={() => setIsModalOpen(false)}
        scriptName={executedScript}
        onViewLogs={handleViewLogs}
      />
    </>
  )
}
```

The modal features a clean design with the script name highlighted in accent color, two action buttons for closing or viewing logs, and uses OpenFrame's design system styling including the DM Sans font family for consistent branding.
