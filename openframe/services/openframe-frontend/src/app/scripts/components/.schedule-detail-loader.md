A loading skeleton component that displays placeholder content while schedule detail data is being fetched, providing a smooth user experience during page transitions.

## Key Components

- **`ScheduleDetailLoader`** - Main export that renders the complete skeleton layout matching the actual schedule detail page structure
- **`InfoBarSkeleton`** - Creates placeholder for the 4-column info bar (Date, Time, Repeat, Platform)
- **`TabsSkeleton`** - Generates skeleton for the 3-tab navigation (Scripts, Devices, History)
- **`TableRowSkeleton`** - Renders individual table row placeholders for tab content

## Usage Example

```typescript
import { ScheduleDetailLoader } from './schedule-detail-loader'

export default function SchedulePage() {
  const { data: schedule, isLoading } = useScheduleQuery(id)

  if (isLoading) {
    return <ScheduleDetailLoader />
  }

  return <ScheduleDetailView schedule={schedule} />
}
```

The component uses the Skeleton component from the UI library and follows responsive design patterns with Tailwind CSS classes. It creates a full-page skeleton that matches the actual layout structure, including back navigation, title section, info cards, tabs, and table rows to provide visual continuity during loading states.
