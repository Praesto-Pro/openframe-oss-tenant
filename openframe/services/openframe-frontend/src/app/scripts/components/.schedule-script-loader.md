A comprehensive loading skeleton component that mirrors the exact layout and structure of the Schedule Script page, providing visual feedback during data fetching with responsive design patterns.

## Key Components

- **`ScriptInfoSectionSkeleton`** - Skeleton for script metadata card with headline, description, and platform details
- **`ScriptArgumentsSkeleton`** - Loading placeholder for script arguments/environment variables sections
- **`DeviceCardSkeleton`** - Individual device selection card skeleton with checkbox, icon, and device info
- **`ScheduleScriptLoader`** - Main skeleton component that combines all sections into a complete page loader

## Usage Example

```typescript
import { ScheduleScriptLoader } from './schedule-script-loader'

export function ScheduleScriptPage({ scriptId }: { scriptId: string }) {
  const { data: script, isLoading } = useScript(scriptId)
  
  if (isLoading) {
    return <ScheduleScriptLoader />
  }
  
  return <ScheduleScriptView script={script} />
}
```

The skeleton provides pixel-perfect loading states for the schedule script interface, including responsive grid layouts for devices (1-4 columns based on screen size), form fields for script configuration, and device/organization filtering sections. Each skeleton element uses consistent spacing and border radius to match the actual UI components.
