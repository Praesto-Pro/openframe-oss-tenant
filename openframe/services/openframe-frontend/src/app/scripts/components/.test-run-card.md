A responsive card component that displays test run information with real-time output logs and status management for device testing operations.

## Key Components

- **TestRunData interface** - Type definition for test run data including device, timing, status, and output
- **TestRunCard component** - Main card component with responsive layout
- **formatDuration()** - Utility function to format elapsed time as HH:MM:SS
- **getStatusLabel() / getStatusColor()** - Helper functions for status display formatting

## Usage Example

```typescript
import { TestRunCard, TestRunData } from './test-run-card'

function TestRunList() {
  const handleStopRun = (runId: string) => {
    // Stop the test run
    console.log(`Stopping run ${runId}`)
  }

  const testRun: TestRunData = {
    id: 'run-123',
    deviceName: 'iPhone 14 Pro',
    agentToolId: 'tool-456',
    startedAt: '2024-01-15 14:30:00',
    startTime: Date.now(),
    status: 'running',
    output: ['Starting test...', 'Device connected', 'Running automation...'],
    elapsedSeconds: 125
  }

  return (
    <TestRunCard
      run={testRun}
      onStop={handleStopRun}
    />
  )
}
```

The component features a responsive layout that adapts across desktop, tablet, and mobile viewports, auto-scrolling log output, and conditional rendering of stop controls for running tests. Status indicators use color coding (accent for running, green for completed, red for errors/aborted).
