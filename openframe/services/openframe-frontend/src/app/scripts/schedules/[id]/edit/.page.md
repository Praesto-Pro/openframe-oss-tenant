A client-side Next.js page component for editing existing schedules in the Flamingo MSP platform, dynamically extracting schedule ID from URL parameters.

## Key Components

- **EditSchedulePage**: Main page component that renders the schedule editing interface
- **Dynamic route configuration**: Forces dynamic rendering to ensure fresh data on each request
- **URL parameter extraction**: Uses Next.js `useParams` hook to get schedule ID from route
- **Layout integration**: Wraps content in the standard AppLayout component

## Usage Example

```typescript
// Route structure: /schedules/edit/[id]/page.tsx
// URL: /schedules/edit/abc123 â†’ scheduleId = "abc123"

// Component renders automatically when navigating to edit route
// The ScheduleCreateView receives the extracted ID and handles edit mode
<AppLayout>
  <ScheduleCreateView scheduleId="abc123" />
</AppLayout>
```

The component leverages Next.js dynamic routing to extract the schedule ID from the URL path and passes it to the `ScheduleCreateView` component, which handles both create and edit functionality based on whether an ID is provided. The `'use client'` directive ensures this component runs on the client side for proper parameter extraction and dynamic behavior.
