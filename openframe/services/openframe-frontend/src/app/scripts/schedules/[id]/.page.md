A Next.js client-side page component that displays detailed information for a specific schedule item, extracting the schedule ID from the URL parameters and rendering it within the application layout.

## Key Components

- **ScheduleDetailPage** - Default exported React component that serves as the main page
- **Dynamic routing** - Configured with `force-dynamic` for server-side rendering on each request
- **URL parameter extraction** - Uses Next.js `useParams` hook to get schedule ID from route
- **Layout composition** - Combines `AppLayout` wrapper with `ScheduleDetailView` content

## Usage Example

This page is typically accessed through Next.js routing at a path like `/schedules/[id]`:

```typescript
// Route: /schedules/abc123
// The component will extract 'abc123' as the schedule ID

function ScheduleDetailPage() {
  const params = useParams<{ id?: string }>()
  const id = typeof params?.id === 'string' ? params.id : ''
  
  // id will be 'abc123'
  return (
    <AppLayout>
      <ScheduleDetailView scheduleId={id} />
    </AppLayout>
  )
}
```

The page ensures type safety by validating that the `id` parameter is a string before passing it to the `ScheduleDetailView` component, falling back to an empty string if the parameter is missing or invalid.
