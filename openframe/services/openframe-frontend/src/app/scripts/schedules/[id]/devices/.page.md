A Next.js page component for assigning devices to a specific schedule, utilizing client-side routing to extract the schedule ID from URL parameters.

## Key Components

- **ScheduleAssignDevicesPage** - Default exported page component that renders the device assignment interface
- **dynamic** - Next.js export configured to force dynamic rendering
- **useParams** - Next.js hook for extracting route parameters
- **AppLayout** - Layout wrapper component for consistent page structure
- **ScheduleAssignDevicesView** - Main view component handling device assignment logic

## Usage Example

This page is accessed via a dynamic route pattern where `id` represents the schedule identifier:

```typescript
// Route: /schedules/[id]/assign-devices
// Example URL: /schedules/abc123/assign-devices

// The component extracts the schedule ID and passes it to the view
function ScheduleAssignDevicesPage() {
  const params = useParams<{ id?: string }>()
  const id = typeof params?.id === 'string' ? params.id : ''
  
  return (
    <AppLayout>
      <ScheduleAssignDevicesView scheduleId={id} />
    </AppLayout>
  )
}
```

The page implements proper type safety by validating the `id` parameter as a string before passing it to the child component, ensuring robust handling of route parameters in the device assignment workflow.
