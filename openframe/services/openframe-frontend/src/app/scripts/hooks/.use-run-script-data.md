A React hook that fetches script details and compatible devices for script execution workflows.

## Key Components

- **`runScriptDataQueryKeys`** - Query key factory for React Query caching
- **`fetchDevicesForScript()`** - API function to fetch devices filtered by supported platforms and online/offline status
- **`useRunScriptData()`** - Main hook that combines script details with compatible devices list
- **`UseRunScriptDataOptions`** - Interface defining the required `scriptId` parameter

## Usage Example

```typescript
import { useRunScriptData } from './use-run-script-data'

function ScriptExecutionPanel({ scriptId }: { scriptId: string }) {
  const {
    scriptDetails,
    devices,
    isLoadingScript,
    isLoadingDevices,
    scriptError,
    devicesError
  } = useRunScriptData({ scriptId })

  if (isLoadingScript || isLoadingDevices) {
    return <div>Loading script and devices...</div>
  }

  if (scriptError || devicesError) {
    return <div>Error: {scriptError || devicesError}</div>
  }

  return (
    <div>
      <h2>Running: {scriptDetails?.name}</h2>
      <p>Compatible devices: {devices.length}</p>
      <ul>
        {devices.map(device => (
          <li key={device.id}>{device.name} - {device.status}</li>
        ))}
      </ul>
    </div>
  )
}
```

The hook automatically filters devices based on the script's supported platforms and only includes online/offline devices, making it ideal for script execution interfaces where you need to show available target devices.
