React Query hooks for managing script schedule data from the TacticalRMM API. Provides type-safe access to schedule lists, details, assigned agents, and execution history with built-in loading states and error handling.

## Key Components

### Query Keys
- **`scriptScheduleQueryKeys`** - Hierarchical query key factory for React Query caching

### Hooks
- **`useScriptSchedules()`** - Fetches list of all script schedules
- **`useScriptSchedule(id)`** - Fetches detailed schedule information by ID
- **`useScriptScheduleAgents(id)`** - Fetches agents assigned to a specific schedule
- **`useScriptScheduleHistory(id, options)`** - Fetches paginated execution history with limit/offset

### Internal API Functions
- **`fetchScriptSchedules()`** - Gets schedule list from tactical API client
- **`fetchScriptSchedule()`** - Gets individual schedule details
- **`fetchScriptScheduleAgents()`** - Gets assigned agents for schedule
- **`fetchScriptScheduleHistory()`** - Gets paginated execution history

## Usage Example

```typescript
import { 
  useScriptSchedules, 
  useScriptSchedule, 
  useScriptScheduleHistory 
} from './use-script-schedule'

function SchedulesList() {
  const { schedules, isLoading, error } = useScriptSchedules()
  
  if (isLoading) return <div>Loading schedules...</div>
  if (error) return <div>Error: {error}</div>
  
  return (
    <ul>
      {schedules.map(schedule => (
        <li key={schedule.id}>{schedule.name}</li>
      ))}
    </ul>
  )
}

function ScheduleDetail({ id }: { id: string }) {
  const { schedule } = useScriptSchedule(id)
  const { history, total } = useScriptScheduleHistory(id, { 
    limit: 10, 
    offset: 0 
  })
  
  return (
    <div>
      <h1>{schedule?.name}</h1>
      <p>Execution History ({total} total)</p>
    </div>
  )
}
```
