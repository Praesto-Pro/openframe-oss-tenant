Utility functions for handling platform compatibility and filtering in script management workflows.

## Key Components

- **HIDDEN_PLATFORMS** - Array of platform IDs that should not be displayed in UI
- **DISABLED_PLATFORMS** - Array of platform IDs that are currently disabled
- **AVAILABLE_PLATFORMS** - Filtered list of platforms available for selection
- **mapPlatformsToOsTypes()** - Maps script platform identifiers to device filter values
- **mapPlatformsForDisplay()** - Maps script platform identifiers to human-readable display names

## Usage Example

```typescript
import { 
  AVAILABLE_PLATFORMS, 
  mapPlatformsToOsTypes, 
  mapPlatformsForDisplay 
} from './script-utils'

// Get platforms available for script targeting
const availablePlatforms = AVAILABLE_PLATFORMS
// Returns platforms excluding 'linux' (hidden)

// Convert script platforms to device filter format
const scriptPlatforms = ['windows', 'darwin']
const osTypes = mapPlatformsToOsTypes(scriptPlatforms)
// Returns: ['WINDOWS', 'MAC_OS']

// Convert platforms for user display
const displayNames = mapPlatformsForDisplay(scriptPlatforms)
// Returns: ['Windows', 'macOS']
```

The utility handles the mapping between different platform naming conventions used across the system - script identifiers (`windows`, `darwin`, `linux`), device filter values (`WINDOWS`, `MAC_OS`), and display names (`Windows`, `macOS`, `Linux`).
