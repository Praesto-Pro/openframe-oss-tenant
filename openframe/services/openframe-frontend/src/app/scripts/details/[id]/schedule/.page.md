A Next.js page component that handles script scheduling functionality with feature flag validation and error handling for disabled features.

## Key Components

- **ScheduleScriptPage** - Main page component that renders the script scheduling interface
- **Feature Flag Check** - Validates if script scheduling is enabled before rendering content
- **Route Parameter Extraction** - Extracts script ID from URL parameters
- **Error Handling** - Shows appropriate error message when feature is disabled

## Usage Example

```typescript
// This page is automatically rendered when navigating to /scripts/[id]/schedule
// URL structure: /scripts/abc123/schedule

// Feature flag configuration in your app
const featureFlags = {
  scriptSchedule: {
    enabled: () => process.env.ENABLE_SCRIPT_SCHEDULE === 'true'
  }
}

// Navigation to this page
router.push(`/scripts/${scriptId}/schedule`)
```

The component uses Next.js App Router conventions with dynamic routing (`[id]`) and includes client-side navigation with the `'use client'` directive. When the script scheduling feature is disabled, users see a friendly error message with an option to return to the scripts list. The component passes the extracted script ID to the `ScheduleScriptView` component for rendering the scheduling interface.
