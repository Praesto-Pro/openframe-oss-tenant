This is the root layout component for the OpenFrame application, providing the foundational structure and metadata configuration for all pages in the Next.js app.

## Key Components

- **Metadata Configuration** - Complete SEO setup with Open Graph, Twitter cards, icons, and manifest
- **Font Setup** - Configures Azeret Mono and DM Sans typography with CSS variables
- **Provider Stack** - Wraps the app with QueryClient, RouteGuard, and authentication providers
- **Runtime Environment** - Loads public environment variables at runtime
- **Analytics Integration** - Google Tag Manager for tracking and analytics
- **Conditional Auth Components** - DevTicketObserver and GraphQL introspection when auth is enabled
- **Global Styling** - Imports global CSS and OpenFrame core styles

## Usage Example

```typescript
// This layout automatically wraps all pages in your Next.js app
// The metadata is applied to all routes unless overridden

// Example page that inherits this layout:
export default function HomePage() {
  return (
    <div>
      <h1>Welcome to OpenFrame</h1>
      {/* This content renders inside the layout's children prop */}
    </div>
  )
}

// The layout provides:
// - SEO metadata for MSP-focused content
// - Dark theme with OpenFrame branding
// - Authentication and route protection
// - Global toast notifications
// - Responsive mobile-optimized viewport
```

The layout enforces dynamic rendering to prevent SSG issues with search parameters and includes comprehensive OpenFrame branding with the tagline "Swap bloated vendor tools for open ones. Automate the boring crap. Take your margin back."
