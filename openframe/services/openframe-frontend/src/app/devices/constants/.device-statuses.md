Defines device status constants and filtered status arrays for the Flamingo MSP platform's device management system. This module provides type-safe status values and pre-configured status collections for different UI contexts.

## Key Components

- **`DEVICE_STATUS`** - Complete enum of all possible device statuses (ONLINE, OFFLINE, ACTIVE, etc.)
- **`DeviceStatus`** - Type derived from all possible status values
- **`DEFAULT_VISIBLE_STATUSES`** - Array of statuses shown in standard device listings (excludes ARCHIVED/DELETED)
- **`DEFAULT_DASHBOARD_STATUSES`** - Minimal status set for dashboard views (ONLINE/OFFLINE only)
- **`HIDDEN_DEVICE_STATUSES`** - Statuses excluded from default views
- **`DefaultVisibleStatus`** and **`HiddenDeviceStatus`** - Derived types for filtered status sets

## Usage Example

```typescript
import { 
  DEVICE_STATUS, 
  DEFAULT_VISIBLE_STATUSES, 
  type DeviceStatus 
} from './device-statuses'

// Filter devices by visible statuses
const visibleDevices = allDevices.filter(device => 
  DEFAULT_VISIBLE_STATUSES.includes(device.status)
)

// Type-safe status checking
function isDeviceOnline(status: DeviceStatus): boolean {
  return status === DEVICE_STATUS.ONLINE
}

// Query with specific status
const maintenanceDevices = await getDevicesByStatus(DEVICE_STATUS.MAINTENANCE)
```

The module uses `as const` assertions and `satisfies` for strict type checking, ensuring status arrays remain synchronized with the main enum while preventing accidental modifications.
