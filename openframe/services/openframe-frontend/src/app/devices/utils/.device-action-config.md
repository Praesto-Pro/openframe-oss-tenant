Centralized configuration system for device action buttons across device tables and detail pages, providing consistent button definitions, availability logic, and platform-specific shell options.

## Key Components

- **DeviceActionButtonConfig**: Interface defining button properties (label, icon, href, disabled state)
- **ShellSubmenuItem**: Configuration for shell type options (CMD, PowerShell, Bash)
- **getDeviceActionButtons()**: Main factory function returning standardized button configurations
- **getWindowsShellSubmenu()**: Returns Windows-specific shell options (CMD, PowerShell)
- **toActionsMenuItem()**: Adapter function converting configs to dropdown menu format
- **getShellHref()**: URL generator for shell sessions with specific shell types

## Usage Example

```typescript
import { getDeviceActionButtons } from './device-action-config'

// Get unified button configurations for a device
const device = { id: '123', os: 'windows', status: 'online' }
const actionButtons = getDeviceActionButtons(device, '123', true)

// Use in dropdown menu
const dropdownItems = [
  toActionsMenuItem(actionButtons.remoteControl, '123'),
  toActionsMenuItem(actionButtons.remoteShell, '123', {
    onShellSelect: (type) => navigate(`/shell?type=${type}`)
  }),
  toActionsMenuItem(actionButtons.manageFiles, '123')
]

// Use individual button configs
const { remoteControl, remoteShell, manageFiles } = actionButtons
```
