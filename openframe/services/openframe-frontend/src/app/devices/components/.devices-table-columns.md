Table column configuration for the OpenFrame device management interface, defining column structure, filtering options, and cell rendering components.

## Key Components

- **`getDeviceTableColumns()`** - Main function that returns an array of table column configurations for device listings
- **`getDeviceTableRowActions()`** - Factory function that creates device action dropdown components for table rows
- **`OrganizationCell`** - React component for rendering organization information with optional logo display
- **Column Definitions** - Four main columns: Device, Status, OS, and Organization with responsive visibility and filtering

## Usage Example

```typescript
import { getDeviceTableColumns, getDeviceTableRowActions } from './devices-table-columns'

// Configure table columns with filtering data
const columns = getDeviceTableColumns({
  statuses: [
    { value: 'ONLINE', label: 'Online' },
    { value: 'OFFLINE', label: 'Offline' }
  ],
  osTypes: [
    { value: 'Windows', label: 'Windows' },
    { value: 'macOS', label: 'macOS' }
  ],
  organizationIds: [
    { value: '123', label: 'Acme Corp' }
  ]
})

// Create row actions with refresh callback
const rowActions = getDeviceTableRowActions(() => {
  console.log('Device action completed, refreshing table')
})

// Use with table component
<Table 
  columns={columns}
  data={devices}
  renderRowActions={rowActions}
/>
```

The component includes responsive design features (hiding columns on smaller screens), status filtering with special handling for archived devices, and integration with the OpenFrame design system components.
