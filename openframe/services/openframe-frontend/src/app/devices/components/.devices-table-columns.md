Defines table column configurations and custom renderers for device data display in OpenFrame's device management interface.

## Key Components

- **`getDeviceTableColumns()`** - Returns table column definitions with filterable options and custom cell renderers
- **`getDeviceTableRowActions()`** - Creates action dropdown renderer for table rows with refresh callback
- **`OrganizationCell`** - Custom cell component displaying organization info with optional icon
- **Device Status Filtering** - Handles status visibility rules (excludes DELETED, separates ARCHIVED)

## Usage Example

```typescript
import { getDeviceTableColumns, getDeviceTableRowActions } from './devices-table-columns'

// Set up table columns with filtering
const columns = getDeviceTableColumns(deviceFilters)

// Create row actions with refresh callback
const rowActions = getDeviceTableRowActions(() => {
  // Refresh table data
  refetchDevices()
})

// Use in table component
<DataTable
  columns={columns}
  data={devices}
  rowActions={rowActions}
/>
```

The columns include device info with type icons, status badges with timestamps, OS type badges, and organization details. Status filtering follows visibility rules where normal statuses appear first, archived items are separated by a divider, and deleted items are completely hidden. Each column supports responsive breakpoints and custom filtering options based on available device data.
