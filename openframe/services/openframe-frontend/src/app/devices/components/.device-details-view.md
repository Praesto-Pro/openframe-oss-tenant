A comprehensive device details view component that displays device information with remote management capabilities and tabbed navigation for different device aspects.

## Key Components

- **Main Container**: `DeviceDetailsView` - Primary component managing device detail display and interactions
- **Data Management**: Uses `useDeviceDetails` hook for fetching device information
- **Action Controls**: Remote control, shell access, file management buttons with OS-specific shell selection
- **Tab System**: Integrated tab navigation for different device information sections
- **Modal Interfaces**: Scripts modal for remote script execution
- **URL Integration**: Handles URL parameters for actions and tab synchronization

## Usage Example

```typescript
import { DeviceDetailsView } from './device-details-view'

// Basic device details page
function DeviceDetailsPage({ params }: { params: { deviceId: string } }) {
  return <DeviceDetailsView deviceId={params.deviceId} />
}

// With URL action handling (automatic script modal opening)
// Navigate to: /devices/details/123?action=runScript
function handleDeviceAction(deviceId: string) {
  router.push(`/devices/details/${deviceId}?action=runScript`)
}
```

The component automatically handles device loading states, error scenarios, and provides platform-specific remote access options (CMD/PowerShell for Windows, Bash for Unix-like systems). It integrates with the broader OpenFrame device management system and supports real-time updates through periodic refresh intervals.
