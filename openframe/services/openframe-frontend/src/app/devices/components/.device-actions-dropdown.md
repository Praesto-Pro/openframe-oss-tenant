A client-side React component that provides a context-aware dropdown menu for device management actions in the OpenFrame platform. It handles remote control, script execution, file management, and device lifecycle operations with integrated confirmation dialogs.

## Key Components

- **DeviceActionsDropdown**: Main component with context-aware action menus
- **Action Handlers**: Functions for remote control, shell access, script execution, and file management
- **Confirmation Dialogs**: Archive and delete confirmation modals with loading states
- **Uninstall Dialog**: Modal showing platform-specific uninstall commands with clipboard copy
- **Menu Groups**: Dynamically generated action groups based on device capabilities and context

## Usage Example

```typescript
import { DeviceActionsDropdown } from './device-actions-dropdown'
import type { Device } from '../types/device.types'

function DeviceTable({ devices }: { devices: Device[] }) {
  return (
    <div>
      {devices.map(device => (
        <div key={device.id} className="flex items-center justify-between">
          <span>{device.displayName}</span>
          {/* Table context - shows all actions in dropdown */}
          <DeviceActionsDropdown
            device={device}
            context="table"
            onActionComplete={() => refetchDevices()}
          />
        </div>
      ))}
    </div>
  )
}

function DeviceDetail({ device }: { device: Device }) {
  return (
    <div className="device-detail">
      {/* Detail context - limited actions in dropdown */}
      <DeviceActionsDropdown
        device={device}
        context="detail"
        onRunScript={() => setShowScriptModal(true)}
        onActionComplete={() => router.push('/devices')}
      />
    </div>
  )
}
```

The component automatically handles device capability detection, platform-specific commands, and navigation routing based on the provided context and device properties.
