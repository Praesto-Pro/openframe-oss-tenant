A Next.js 14 page component that provides remote desktop access to devices through MeshCentral WebRTC tunneling, featuring real-time connection management, power controls, and display switching capabilities.

## Key Components

- **RemoteDesktopPage**: Main page component handling device connection and desktop streaming
- **MeshControlClient**: WebSocket client for MeshCentral authentication and control commands
- **MeshTunnel**: WebRTC tunnel implementation for binary data streaming
- **MeshDesktop**: Canvas-based desktop rendering with input handling
- **RemoteSettingsModal**: Configuration modal for desktop display settings
- **Display Management**: Multi-monitor support with dynamic display switching
- **Power Controls**: Remote wake/sleep/reset/poweroff functionality
- **Keyboard Shortcuts**: Predefined key combinations and Ctrl+Alt+Del support

## Usage Example

```typescript
// Access remote desktop for a device
// Route: /devices/[deviceId]/remote-desktop

// With legacy device data (backward compatibility)
const url = `/devices/123/remote-desktop?deviceData=${JSON.stringify({
  id: '123',
  meshcentralAgentId: 'node-abc123',
  hostname: 'workstation-01',
  organization: 'Acme Corp'
})}`

// Modern usage - fetches device data internally
router.push(`/devices/123/remote-desktop`)

// Component handles:
// 1. Device data resolution (legacy vs fetched)
// 2. MeshCentral tunnel establishment  
// 3. Canvas-based desktop streaming
// 4. Input forwarding and power management
// 5. Multi-display support and settings
```

The component supports both legacy query parameter device data and modern internal fetching, automatically manages connection states with reconnection logic, and provides comprehensive remote desktop functionality through MeshCentral's WebRTC infrastructure.
