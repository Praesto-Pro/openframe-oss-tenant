A Next.js page component that provides file management functionality for devices in the Flamingo OpenFrame platform, enabling users to browse and manage files on connected devices through MeshCentral integration.

## Key Components

- **FileManagerPage** - Main page component that handles device file management
- **FileManagerPageSkeleton** - Loading state component with skeleton UI
- **useDeviceDetails** - Custom hook for fetching device information
- **FileManagerContainer** - Core file management interface component
- **getMeshCentralAgentId** - Utility function to extract MeshCentral agent ID

## Usage Example

```typescript
// This page is automatically rendered by Next.js routing
// URL: /devices/details/[deviceId]/file-manager

// Example of the data flow:
const deviceId = "device-123"
const { deviceDetails, isLoading, error } = useDeviceDetails()

// Component renders different states:
if (isLoading) return <FileManagerPageSkeleton />
if (error) return <ErrorMessage />
if (!meshcentralAgentId) return <RequiresAgentMessage />

// When ready, renders the file manager:
return (
  <FileManagerContainer
    deviceId={deviceId}
    meshcentralAgentId={meshcentralAgentId}
    hostname={deviceDetails.hostname}
  />
)
```

The page includes comprehensive error handling for missing MeshCentral agents, loading states with skeleton UI, and proper navigation back to device details. It extracts the device ID from URL parameters and validates that the device has the required MeshCentral integration before enabling file management functionality.
