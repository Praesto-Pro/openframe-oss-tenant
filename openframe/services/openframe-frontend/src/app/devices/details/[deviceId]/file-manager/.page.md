A Next.js page component that provides a file manager interface for remote device management through MeshCentral integration. The page handles device parameter resolution, loading states, error conditions, and renders the file manager when all prerequisites are met.

## Key Components

- **FileManagerPage** - Main page component that manages device data fetching and conditional rendering
- **FileManagerPageSkeleton** - Loading state component with back navigation and skeleton placeholders
- **useDeviceDetails** - Custom hook for fetching and managing device information
- **FileManagerContainer** - Core file manager component that handles the actual file operations
- **getMeshCentralAgentId** - Utility function to extract MeshCentral agent identifier from device data

## Usage Example

```typescript
// This page is automatically rendered by Next.js routing at:
// /devices/details/[deviceId]/file-manager

// The component expects a deviceId parameter and will:
// 1. Fetch device details using the provided ID
// 2. Extract the MeshCentral agent ID for remote file operations
// 3. Render the file manager interface with proper error handling

// Error states handled:
// - Loading: Shows FileManagerPageSkeleton
// - Fetch error: Shows error message with back navigation
// - Missing MeshCentral ID: Shows specific error for missing agent
```

The page implements comprehensive error handling and loading states, ensuring users can always navigate back to the device details view. It requires a valid MeshCentral agent ID to enable remote file management functionality.
