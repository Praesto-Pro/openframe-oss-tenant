Centralized constants and type definitions for the dialog/chat system, defining all status enums, API endpoints, and configuration values used across the OpenFrame chat platform.

## Key Components

### Dialog Management
- **DIALOG_STATUS** - Dialog states (ON_HOLD, RESOLVED, ACTIVE)
- **CHAT_TYPE** - Chat contexts (CLIENT, ADMIN)
- **MESSAGE_TYPE** - Message categories including tool execution and approval workflows
- **OWNER_TYPE** - Message ownership (CLIENT, ADMIN, ASSISTANT)

### Assistant Configuration
- **ASSISTANT_CONFIG** - AI assistant definitions (Fae for clients, Mingo for admins)
- **APPROVAL_STATUS** - Approval workflow states (pending, approved, rejected)

### Infrastructure
- **API_ENDPOINTS** - REST API routes for chat operations
- **NATS_TOPICS** - Message queue topics for real-time communication
- **STORAGE_KEYS** - Local storage keys for authentication
- **NETWORK_CONFIG** - Connection timeouts, retry intervals, and polling limits

## Usage Example

```typescript
import { DIALOG_STATUS, MESSAGE_TYPE, ASSISTANT_CONFIG } from './index'

// Check dialog state
if (dialog.status === DIALOG_STATUS.ACTIVE) {
  // Process active dialog
}

// Create message with proper typing
const message = {
  type: MESSAGE_TYPE.EXECUTING_TOOL,
  owner: OWNER_TYPE.ASSISTANT,
  content: 'Running diagnostic...'
}

// Configure assistant
const assistant = ASSISTANT_CONFIG.FAE.type // 'fae'
const assistantName = ASSISTANT_CONFIG.MINGO.name // 'Mingo'
```
