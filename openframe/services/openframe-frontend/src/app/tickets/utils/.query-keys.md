A centralized query key factory for managing React Query cache keys and invalidation for ticket/dialog data fetching operations.

## Key Components

- **`DialogsQueryParams`** - Interface defining filter parameters for dialog queries including archived status, search terms, status filters, and pagination cursor
- **`dialogsQueryKeys`** - Hierarchical query key factory with structured cache keys for different dialog query types
- **`invalidateAllDialogs`** - Utility function to invalidate all dialog-related queries in the React Query cache

## Usage Example

```typescript
import { useQuery, useQueryClient } from '@tanstack/react-query'
import { dialogsQueryKeys, invalidateAllDialogs, DialogsQueryParams } from './query-keys'

// Using query keys in a React Query hook
const useDialogs = (params: DialogsQueryParams) => {
  return useQuery({
    queryKey: dialogsQueryKeys.list(params),
    queryFn: () => fetchDialogs(params)
  })
}

// Invalidating all dialog queries after a mutation
const queryClient = useQueryClient()
const createDialog = useMutation({
  mutationFn: (data) => createNewDialog(data),
  onSuccess: () => {
    invalidateAllDialogs(queryClient)
  }
})

// Example query parameters
const queryParams: DialogsQueryParams = {
  archived: false,
  search: 'bug report',
  statusFilters: ['open', 'in-progress'],
  cursor: 'page-2'
}
```

This pattern ensures consistent cache key generation and makes query invalidation predictable across the application.
