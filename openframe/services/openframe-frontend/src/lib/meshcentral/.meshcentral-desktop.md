MeshCentral desktop client that provides remote desktop access with mouse, keyboard, and display management capabilities through a WebSocket-based protocol.

## Key Components

- **`DisplayInfo`** - Interface defining display configuration with dimensions, position, and primary status
- **`DesktopInputHandlers`** - Type defining the contract for desktop input handling methods
- **`MeshDesktop`** - Main class implementing remote desktop functionality with canvas rendering and input capture

## Usage Example

```typescript
import { MeshDesktop } from './meshcentral-desktop'

const desktop = new MeshDesktop()
const canvas = document.getElementById('desktop-canvas') as HTMLCanvasElement

// Set up WebSocket sender for protocol communication
desktop.setSender((data: Uint8Array) => {
  websocket.send(data)
})

// Attach to canvas and configure
desktop.attach(canvas)
desktop.setViewOnly(false)
desktop.setSwapMouseButtons(false)

// Handle display changes
desktop.onDisplayListChange?.((displays) => {
  console.log('Available displays:', displays)
})

// Handle incoming data from WebSocket
websocket.onmessage = (event) => {
  desktop.processMessage(new Uint8Array(event.data))
}

// Cleanup
desktop.detach()
```

The class handles MeshCentral's binary protocol for desktop streaming, including mouse/keyboard input encoding, canvas rendering, multi-display support, and connection lifecycle management. It provides configurable options for view-only mode, mouse button swapping, and scroll direction inversion.
