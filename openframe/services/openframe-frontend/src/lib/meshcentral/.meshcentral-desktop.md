TypeScript implementation of a MeshCentral desktop remote control client that handles canvas-based desktop streaming with mouse and keyboard input management.

## Key Components

- **`DisplayInfo`** - Interface defining display properties (id, position, dimensions, primary status)
- **`DesktopInputHandlers`** - Interface defining required methods for desktop input handling
- **`MeshDesktop`** - Main class implementing remote desktop functionality with:
  - Canvas attachment/detachment for rendering
  - Mouse event handling (move, click, wheel, double-click) 
  - Keyboard event handling with virtual key mapping
  - Binary protocol encoding for MeshCentral communication
  - Multi-display support and display switching
  - Tile-based image rendering with decode queue management

## Usage Example

```typescript
// Initialize desktop controller
const desktop = new MeshDesktop();

// Attach to canvas element
const canvas = document.getElementById('desktop-canvas') as HTMLCanvasElement;
desktop.attach(canvas);

// Set up WebSocket sender for MeshCentral protocol
desktop.setSender((data: Uint8Array) => {
  websocket.send(data);
});

// Configure view-only mode
desktop.setViewOnly(false);

// Handle display changes
desktop.onDisplayListChange?.((displays) => {
  console.log('Available displays:', displays);
});

// Switch to different display
desktop.switchDisplay?.(1);

// Send special key combinations
desktop.sendCtrlAltDel?.();
desktop.sendKeyCombo?.('ctrl+alt+f4');
```

The class automatically handles MeshCentral protocol initialization, compression settings, and maintains proper key/mouse state tracking for reliable remote desktop control.
