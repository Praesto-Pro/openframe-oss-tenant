A WebSocket client for controlling MeshCentral instances, providing authentication, tunnel management, and device power actions through the MeshCentral control API.

## Key Components

### MeshControlClient Class
- **WebSocket Management**: Handles connection lifecycle with automatic reconnection
- **Authentication**: Manages auth cookies and session establishment
- **Tunnel Operations**: Creates desktop and file tunnels to managed devices
- **Power Actions**: Performs remote power operations (wake, sleep, reset, poweroff)
- **Request Tracking**: Manages pending requests with timeout handling

### Core Methods
- `openSession()` - Establishes authenticated WebSocket connection
- `getAuthCookies()` - Retrieves authentication cookies from MeshCentral
- `sendDesktopTunnel()` - Creates desktop remote control tunnels
- `sendFileTunnel()` - Creates file transfer tunnels
- `powerAction()` - Performs power operations on devices
- `reconnect()` - Reconnects with fresh authentication

## Usage Example

```typescript
import { MeshControlClient } from './meshcentral-control'

// Initialize client with credentials
const client = new MeshControlClient({
  user: 'admin',
  pass: 'password'
})

// Open authenticated session
await client.openSession()

// Create desktop tunnel for remote control
client.sendDesktopTunnel(
  'node123', // nodeId
  'relay456', // relayId
  undefined,  // relayCookie (optional)
  'domain'    // domainPrefix (optional)
)

// Perform power action
await client.powerAction('node123', 'reset')

// Check connection status
if (client.isConnected()) {
  console.log('MeshCentral control connection active')
}

// Clean up
client.close()
```

The client automatically handles WebSocket reconnection, maintains active tunnel state, and provides promise-based APIs for asynchronous operations.
