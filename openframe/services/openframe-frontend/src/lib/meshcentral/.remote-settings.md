Configuration management for MeshCentral remote desktop sessions, handling quality, scaling, frame rate, and input settings with binary protocol communication.

## Key Components

- **RemoteSettingsConfig**: Interface defining all remote desktop configuration options
- **QUALITY_OPTIONS**: Predefined JPEG quality levels from 10% to 100%
- **SCALING_OPTIONS**: MeshCentral scaling values mapped to percentages (25%-200%)
- **FRAME_RATE_OPTIONS**: Frame rate presets with corresponding millisecond intervals
- **RemoteDesktopSettings**: Main class for managing settings and WebSocket communication
- **DEFAULT_SETTINGS**: Balanced default configuration for optimal performance

## Usage Example

```typescript
// Initialize with default settings
const remoteSettings = new RemoteDesktopSettings()

// Configure custom settings
const customSettings: RemoteSettingsConfig = {
  quality: 80,
  scaling: 768,  // 75% scaling
  frameRate: 'fast',
  swapMouseButtons: true,
  useRemoteKeyboardMap: false,
  invertScrollDirection: false
}

// Set WebSocket connection
remoteSettings.setWebSocket(websocket)

// Apply settings via binary protocol
remoteSettings.applySettings(customSettings)

// Estimate bandwidth usage
const estimatedKBps = remoteSettings.estimateBandwidth()
console.log(`Estimated bandwidth: ${estimatedKBps} KB/s`)

// Get current settings
const currentSettings = remoteSettings.getSettings()
```

The class handles MeshCentral's binary protocol requirements, including big-endian byte ordering and specific command formatting for settings transmission.
