A centralized runtime configuration utility for Next.js applications that provides type-safe access to environment variables with fallback values and cross-environment compatibility.

## Key Components

- **getEnvVar()** - Internal helper function that safely retrieves environment variables using `next-runtime-env` with fallbacks to browser and Node.js environments
- **runtimeEnv** - Main configuration object exposing typed methods for accessing specific environment variables

### Configuration Methods

- **tenantHostUrl()** - Tenant application host URL
- **sharedHostUrl()** - Shared services host URL  
- **gtmContainerId()** - Google Tag Manager container ID (optional)
- **appMode()** - Application mode (defaults to 'oss-tenant')
- **appType()** - Application type (defaults to 'openframe-dashboard')
- **appUrl()** - Production application URL
- **devUrl()** - Development server URL
- **enableDevTicketObserver()** - Development ticket observer feature flag
- **authCheckIntervalMs()** - Authentication check interval in milliseconds
- **authLoginUrl()** - Authentication login URL
- **featureOrganizationImages()** - Organization images feature flag
- **featureSSOAllowDomain()** - SSO domain allowlist feature flag
- **featureScriptSchedule()** - Script scheduling feature flag

## Usage Example

```typescript
import { runtimeEnv } from './runtime-config'

// Get configuration values
const tenantUrl = runtimeEnv.tenantHostUrl()
const isDevMode = runtimeEnv.appMode() === 'development'
const authInterval = runtimeEnv.authCheckIntervalMs()

// Use in application logic
if (runtimeEnv.featureSSOAllowDomain()) {
  // Enable SSO domain validation
}

// Configure GTM if container ID exists
const gtmId = runtimeEnv.gtmContainerId()
if (gtmId) {
  initializeGTM(gtmId)
}
```
