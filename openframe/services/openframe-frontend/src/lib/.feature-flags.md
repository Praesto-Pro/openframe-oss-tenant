Centralized feature flags system that controls feature availability across the application based on runtime environment configuration.

## Key Components

- **featureFlags** - Main feature flags object with methods to check feature availability
  - `organizationImages` - Controls organization image/logo upload and display functionality
  - `ssoAutoAllow` - Controls SSO automatic domain allowance feature
- **FeatureFlagKey** - Type definition for valid feature flag keys

## Usage Example

```typescript
import { featureFlags } from './feature-flags'

// Check if organization images feature is enabled
if (featureFlags.organizationImages.enabled()) {
  // Show organization image upload UI
  renderImageUpload()
}

// Check specific organization image capabilities
const canUpload = featureFlags.organizationImages.uploadEnabled()
const canDisplay = featureFlags.organizationImages.displayEnabled()

// Check SSO auto-allow feature
if (featureFlags.ssoAutoAllow.enabled()) {
  // Enable automatic domain allowance for SSO
  enableAutoSSO()
}

// Type-safe feature flag key usage
type FlagKey = FeatureFlagKey // 'organizationImages' | 'ssoAutoAllow'
```

The feature flags are determined by the underlying `runtimeEnv` configuration, providing a consistent interface for conditional feature rendering throughout the application.
