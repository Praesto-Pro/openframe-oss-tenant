Provides centralized feature flag management for controlling application features based on runtime environment configuration.

## Key Components

- **`featureFlags`** - Main configuration object containing all feature flag definitions
- **`organizationImages`** - Controls organization image/logo display and upload functionality
- **`ssoAutoAllow`** - Manages SSO auto-allow domain feature availability
- **`scriptSchedule`** - Controls script scheduling feature access
- **`FeatureFlagKey`** - Type definition for valid feature flag keys

Each feature flag provides `enabled()` methods that check the corresponding runtime environment configuration.

## Usage Example

```typescript
import { featureFlags } from './feature-flags'

// Check if organization images are enabled
if (featureFlags.organizationImages.enabled()) {
  // Show organization logo upload UI
  renderImageUpload()
}

// Check specific organization image capabilities
if (featureFlags.organizationImages.uploadEnabled()) {
  enableUploadButton()
}

// Check SSO auto-allow feature
if (featureFlags.ssoAutoAllow.enabled()) {
  allowDomainAutoSSO()
}

// Check script scheduling availability
if (featureFlags.scriptSchedule.enabled()) {
  showSchedulingOptions()
}
```

The feature flags automatically reflect the current runtime environment settings, providing a clean API for conditional feature rendering throughout the application.
