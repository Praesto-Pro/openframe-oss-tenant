Centralized API client that handles both cookie-based and header-based authentication with automatic token refresh, request queuing, and comprehensive error handling for OpenFrame applications.

## Key Components

- **`ApiClient`** - Main singleton class providing authenticated HTTP requests
- **`apiClient`** - Pre-configured singleton instance ready for use
- **`ApiResponse<T>`** - Standardized response interface with data, error, status, and ok fields
- **`ApiRequestOptions`** - Extended fetch options with auth skipping capability
- **Authentication Methods** - Automatic Bearer token injection and cookie-based auth support
- **Token Refresh Logic** - Handles 401 errors with automatic token refresh and request queuing
- **HTTP Convenience Methods** - `get()`, `post()`, `put()`, `patch()`, `delete()` wrappers

## Usage Example

```typescript
import { apiClient } from './api-client'

// Basic GET request
const userResponse = await apiClient.get('/api/users/123')
if (userResponse.ok) {
  console.log('User:', userResponse.data)
}

// POST with body
const createResponse = await apiClient.post('/api/users', {
  name: 'John Doe',
  email: 'john@example.com'
})

// Skip authentication for public endpoints
const publicResponse = await apiClient.get('/api/public/status', {
  skipAuth: true
})

// External API call
const externalResponse = await apiClient.external('https://api.github.com/user')

// Direct user info endpoint
const meResponse = await apiClient.me()
```

The client automatically handles authentication through DevTicket tokens or cookies, refreshes expired tokens, queues requests during refresh operations, and provides unified error handling with automatic logout on authentication failures.
